# Article Prompts

Use together with `00-shared-style-and-constraints.md`.

## Prompt A - Article Desktop

Create a low-fidelity desktop wireframe for a long-form technical article page focused on readability.

Required structure from top to bottom:

1. Top header with Logo, Blog, Topics, About, Search.
2. Article title block.
3. Metadata row with placeholders: Date, Read Time, Topic, Updated.
4. Key summary box with 2-3 bullet placeholders.
5. Main reading area in two columns:
   - Left wide content column: heading blocks, paragraph blocks, code block placeholders, callout blocks, image/diagram placeholders.
   - Right narrow sticky TOC rail: section links and progress indicator placeholder.
6. Related content matrix near bottom with three columns:
   - Next in topic.
   - Adjacent topic.
   - Foundation content.
7. Utility row: Back to topic hub and Next article.
8. Footer block.

Layout guidance:

- Reading column must be clearly dominant.
- TOC rail must look secondary but persistent.
- Use generous spacing between content sections.

Image framing:

- Desktop browser-like frame, landscape orientation.

## Prompt B - Article Mobile

Create a low-fidelity mobile wireframe for the same long-form article page.

Required structure from top to bottom:

1. Mobile header with Logo, Menu icon, Search icon.
2. Article title.
3. Stacked metadata row.
4. Collapsible summary box placeholder.
5. Collapsible TOC module.
6. Single-column article body with headings, paragraph blocks, callouts, and code block placeholders.
7. Related content section with stacked cards.
8. Utility row: Back to topic hub and Next article.
9. Footer block.

Layout guidance:

- Prioritize reading flow and section rhythm.
- Keep modules ordered for progressive discovery.
- Minimize chrome and maximize content area.

Image framing:

- Smartphone portrait frame.

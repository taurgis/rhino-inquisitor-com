# Article Design Prompts

Use together with 00-shared-brand-and-style.md.

## Prompt A - Article Desktop Design Example

Create a high-fidelity desktop design concept for a long-form technical article page.

Structure from top to bottom must follow this order:

1. Header with logo, nav, search.
2. Article title block.
3. Metadata row with date, reading time, topic, and updated info.
4. Summary box with 2-3 key takeaway bullets.
5. Main content area in two columns:
   - Left dominant reading column with headings, paragraphs, code block style, callout style, and image/diagram examples.
   - Right narrow sticky table-of-contents rail with section links and reading progress cue.
6. Related content matrix with three groups:
   - Next in topic.
   - Adjacent topic.
   - Foundation content.
7. Utility row with Back to topic hub and Next article actions.
8. Footer.

Design intent:

- Reading comfort is the primary goal.
- Strong typographic rhythm and generous whitespace.
- TOC rail should feel supportive, not dominant.
- Code and callout blocks should look polished and production-feasible.

Framing:

- Desktop browser frame.
- Landscape composition.

## Prompt B - Article Mobile Design Example

Create a high-fidelity mobile design concept for the same long-form article page.

Structure from top to bottom must follow this order:

1. Mobile header with logo, menu, search.
2. Article title.
3. Stacked metadata row.
4. Collapsible summary box module.
5. Collapsible TOC module.
6. Single-column article body with heading hierarchy, paragraph blocks, code block style, and callout style.
7. Related content section with stacked cards.
8. Utility row with Back to topic hub and Next article.
9. Footer.

Design intent:

- Prioritize focused reading flow.
- Minimize interface chrome and keep content dominant.
- Preserve consistency with desktop style while adapting for touch.

Framing:

- Smartphone portrait frame.
- Full page visible.

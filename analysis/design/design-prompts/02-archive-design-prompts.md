# Archive Design Prompts

Use together with 00-shared-brand-and-style.md.

## Prompt A - Archive Desktop Design Example

Create a high-fidelity desktop design concept for the blog archive page.

Structure from top to bottom must follow this order:

1. Header with logo, nav, search.
2. Archive title plus one-line descriptor.
3. Search and controls row (search input, filter controls, sort control).
4. Main content area in two columns:
   - Left: filter rail (Topic, Type, Difficulty, Year).
   - Right: article card listing (grid or list) with repeating cards.
5. Pagination row (Prev, page numbers, Next) plus jump-to-year control.
6. Footer.

Card requirements:

- Each card should include image thumbnail, title, metadata row, and short excerpt.
- Metadata should visually support fast scanning.

Design intent:

- Discovery-first but still elegant and editorial.
- Filters must be visible and easy to understand.
- Reading hierarchy should prioritize titles and excerpts.

Framing:

- Desktop browser frame.
- Landscape composition.

## Prompt B - Archive Mobile Design Example

Create a high-fidelity mobile design concept for the same archive page.

Structure from top to bottom must follow this order:

1. Mobile header with logo, menu, search.
2. Archive title and descriptor.
3. Search bar.
4. Filter chips row and sort dropdown/toggle.
5. Stacked article cards.
6. Pagination controls.
7. Jump-to-year control.
8. Footer.

Design intent:

- Keep all discovery controls visible near top.
- Make cards compact but readable.
- Preserve single-column rhythm and reduce clutter.

Framing:

- Smartphone portrait frame.
- Full page visible.
